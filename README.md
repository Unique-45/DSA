# Data Structures and Algorithms (DSA)

A comprehensive repository containing implementations of fundamental data structures and algorithms in C.

## 📚 Repository Overview

This repository serves as a complete learning resource and reference guide for understanding core computer science concepts. It covers a wide range of data structures and algorithms with detailed implementations, comments, complexity analysis, and executable examples.

## 📁 Directory Structure

```
DSA/
├── Sorting/                    # Sorting algorithms
├── Searching/                  # Searching algorithms
├── Trees/                      # Tree data structures
├── Graphs/                     # Graph algorithms
├── LinkedLists/                # Linked list implementations
├── Stacks/                     # Stack implementations
├── Queues/                     # Queue implementations
├── DynamicProgramming/         # Dynamic programming problems
├── Recursion/                  # Recursion examples
├── Arrays/                     # Array-based problems
└── README.md                   # This file
```

## 📖 Topics Covered

This repository includes implementations for:
- **Data Structures**: Arrays, Linked Lists, Stacks, Queues, Trees, Graphs, Hash Tables
- **Sorting Algorithms**: Various sorting techniques with complexity analysis
- **Searching Algorithms**: Binary search, linear search, and more
- **Graph Algorithms**: Traversals, shortest path, minimum spanning tree
- **Dynamic Programming**: Optimization problems and solutions
- **Recursion**: Recursive problem solving
- And much more...

## 🚀 Getting Started

### Prerequisites
- GCC compiler (or any C compiler)
- Basic knowledge of C programming
- Understanding of fundamental data structure concepts

### Compilation

Navigate to any algorithm directory and compile:

```bash
cd <topic_directory>/

# Compile
gcc algorithm_name.c -o algorithm_name

# Run
./algorithm_name
```

### General Example Usage

Most programs follow a similar pattern:
1. Input the size or parameters
2. Enter the data elements
3. View the original data
4. See the processed/sorted/result data
5. Performance metrics (when applicable)

## 💡 Key Concepts

- **Time Complexity**: Measure of how the algorithm's runtime grows with input size
- **Space Complexity**: Measure of how much extra memory the algorithm requires
- **Big O Notation**: Standard notation for describing algorithm efficiency
- **Data Structures**: Organized ways to store and access data
- **Algorithms**: Step-by-step procedures for solving problems

## 📊 Complexity Reference

Common time complexities from fastest to slowest:
- O(1) - Constant
- O(log n) - Logarithmic
- O(n) - Linear
- O(n log n) - Linearithmic
- O(n²) - Quadratic
- O(n³) - Cubic
- O(2ⁿ) - Exponential
- O(n!) - Factorial
